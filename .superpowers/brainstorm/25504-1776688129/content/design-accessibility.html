Component Hierarchy
  • <AssessmentLayout /> (The Shell)
  •   ↳ <VisualWrapper /> (Theme/Accessibility)
  •     ↳ <ActiveTest /> (Logic Only)
  •     ↳ <AvatarLayer /> (Future: Hidden)

Theme can be swapped via CSS variables or a React Context provider.

Strategy for Flexibility

  • CSS Variables: Control font size and contrast centrally.
  • Layout Slots: Reservation of "slots" in the UI where an avatar or audio control component can be injected later without refactoring the test logic.
  • Standardized IO: Every test reports a simple JSON result, keeping the "Game Logic" independent of the "Display Layer".
